# Dancify — Functional Test Cases Checklist
App Version: Pre-Release | Platform: iOS (Expo/React Native) | QA Standard: SDET / App Store Submission

================================================================================
Module 1: App Launch & Initialization
================================================================================

Happy Path
----------
[ ] TC-INIT-01 — Cold launch: App opens without crash. Root layout initializes SQLite DB (dancewithme.db) in WAL mode successfully.
[ ] TC-INIT-02 — Fonts load: All Ionicons / Feather vector icons render correctly; no missing glyph boxes or placeholder rectangles visible.
[ ] TC-INIT-03 — i18n initializes: Language detected from device locale on first install; no "translation key fallback" strings (raw keys like `library.import`) visible anywhere in the UI.
[ ] TC-INIT-04 — Tabs render: Both the "Library" tab and "Profile" tab are accessible; active tab indicator highlights the correct tab.
[ ] TC-INIT-05 — Re-launch with existing data: Killing and re-opening the app loads all previously imported videos correctly from SQLite.

Edge Cases & Negative Paths
-----------------------------
[ ] TC-INIT-06 — DB migration on update: After an over-the-air update (or fresh build over existing install), existing video records and settings survive. No DB corruption or crash.
[ ] TC-INIT-07 — Extremely fast repeated launches: Force-quit and relaunch 5 times in rapid succession. App never gets stuck in a blank/frozen state.
[ ] TC-INIT-08 — Low storage on first launch: With < 100MB of device storage free, app launches without crash. No silent DB write failure.
[ ] TC-INIT-09 — Device locale unsupported: Set device language to an unsupported locale (e.g., Japanese). App falls back gracefully to English with no raw translation keys displayed.

Module 2: Onboarding & Language Selection
================================================================================

Happy Path
----------
[ ] TC-LANG-01 — Language modal opens: Tapping the language preference row in the Profile tab navigates to the language selection modal without delay.
[ ] TC-LANG-02 — Switch EN → ZH: Selecting "简体中文" immediately re-renders all visible UI strings in Chinese. Navigate to Library tab; all labels are in Chinese.

Edge Cases & Negative Paths
-----------------------------

[ ] TC-LANG-09 — All translation keys are present in both locales: No screen in either EN or ZH shows a raw i18n key string (e.g., `player.speed`). Verify all 74 keys in both en.json and zh.json.

[ ] TC-IMP-13 — Import a very large video (> 500MB, 4K/60fps): Select a large video file. Compression runs to completion without a memory crash or OOM error. Progress bar advances smoothly.
[ ] TC-IMP-14 — Import a very short video (< 2s): Import a 1-second clip. App handles edge case duration. Thumbnail extracted. Player opens without crash.
[ ] TC-IMP-15 — Duplicate filename deduplication: Import the same video twice. Second entry should be named 视频YYYY-MM-DD (2), not silently overwrite.
[ ] TC-IMP-16 — Background to foreground during compression: Start importing a large video, immediately background the app (home swipe), wait 10s, return to foreground. Compression resumed or completed; progress UI is accurate.
[ ] TC-IMP-17 — Force quit during active compression: Kill the app mid-compression. Re-launch: no orphaned pending card stuck at a partial progress state. No partial/corrupt file remains in the videos/ directory.
[ ] TC-IMP-18 — Storage full during import: Simulate near-full device storage (< 50MB free). App does not crash; it either completes with an error message or handles gracefully without leaving orphaned files.
[ ] TC-IMP-19 — Import two videos simultaneously: Trigger imports back-to-back before the first finishes. Both pending cards show independent progress. Both complete correctly without corrupting each other.
[ ] TC-IMP-20 — Thumbnail extraction failure: On a video format that may not produce a thumbnail (edge case codec), pending card falls back to a placeholder gracefully; app does not crash.
